# [Neg] Регистрация пользователя с валидными данными (sql-инъекция)

## Метаданные

**ID:** 12
**Приоритет:** Средний
**Статус:** Готов
**Длительность:** 0h 10m 0s

## Тестовые данные
- email: `user@example.org`;
- `password: password; DROP TABLE users;--`;

## Предусловия

1. В системе отсутствует пользователь с email %email и паролем %password

## Шаги и ожидаемые результаты

1. **Шаг:** Отправить запрос POST /api/v1/auth/register с телом: {“email“: “%email”, “password“: “%password”}
   **Ожидаемый результат:** Сервер отправляет ответ со статусом 201 Created и сообщением с данными созданного пользователя
2. **Шаг:** Проверить наличие записи в базе данных
   **Ожидаемый результат:** В базе данных создана новая запись с указанным email
3. **Шаг:** Проверить то, что пароль захэширован и не хранится в базе данных в чистом виде
   **Ожидаемый результат:** В базе данных записан хэш пароля
4. **Шаг:** Проверить то, что sql-инъекция не удалила таблицу
   **Ожидаемый результат:** SQL-инъекция не изменила состояния базы данных, таблица users не была удалена
