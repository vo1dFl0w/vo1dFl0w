# [Pos] Вход в систему зарегистрированного пользователя с валидными данными

## Метаданные

**ID:** 14
**Приоритет:** Средний
**Статус:** Готов
**Оценка:** 0h 10m 0s

## Тестовые данные
- email: `user@example.org`;
- password: `password`;

## Предусловия
- Пользователь с email `%email` и паролем `%password` зарегистрирован и существует в базе данных

## Шаги и ожидаемые результаты
1. **Шаг:** Отправить запрос POST /api/v1/auth/login с телом: {“email“: “`%email`“, “password“: “`%password`”}
   **Ожидаемый результат:** Сервер отправляет ответ со статусом 200 OK и телом ответа, содержащим поле `“access_token”: “<token>“`
2. **Шаг:** Проверить то, что в теле ответа нет поля `refresh_token`
   **Ожидаемый результат:** В теле ответа отсутствует поле `refresh_token`
3. **Шаг:** Проверить заголовок Set-Cookie с установленным значением `refresh_token`
   **Ожидаемый результат:** В cookie есть заголовок с полученным `refresh_token`
4. **Шаг:** Проверить наличие записи в базе данных
   **Ожидаемый результат:** В базе данных создана новая запись с новым refresh token
5. **Шаг:** Проверить то, что refresh token хранится в базе данных в захэшированном виде
   **Ожидаемый результат:** В базе данных хранится хэш refresh token
