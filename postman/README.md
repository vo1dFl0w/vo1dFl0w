# Postman Collection for Auth Service

Коллекция Postman для тестирования REST API сервиса авторизации.

## Описание

В коллекции на данный момент собраны запросы и проверки для оценариев регистрации, аутентификации и авторизации:

- регистрация пользователя;
- вход в систему;
- получение информации о текущем пользователе;
- негативные сценарии;
- проверки на безопасность и корректность хранения данных.

## Что покрывает коллекция

### Позитивные сценарии
- регистрация пользователя с валидными данными;
- вход зарегистрированного пользователя с валидными данными;
- получение данных текущего пользователя.

### Негативные сценарии
- регистрация с уже существующим email;
- вход с невалидным email;
- доступ к информации о пользователе без авторизации.

### Проверки безопасности
- проверка того, что пароль не возвращается в ответе;
- проверка, что refresh token не возвращается в теле ответа;
- проверка хранения refresh token в базе данных в захэшированном виде;
- проверка устойчивости к SQL-инъекции.

## Структура коллекции

Коллекция организована по папкам:

- `positive`
- `negative`

Такое разбиение помогает быстро находить нужные тесты и поддерживать порядок в коллекции.

## Используемые эндпоинты

- `POST /api/v1/auth/register`
- `POST /api/v1/auth/login`
- `GET /api/v1/auth/me`

## Требования

Для запуска коллекции потребуется:

- Postman;
- Git;
- Docker.

Запуск [auth-service](https://github.com/vo1dFl0w/auth-service) и отправка запросов Postman:
1. `git clone https://github.com/vo1dFl0w/auth-service`;
2. Находясь в корневой папке, поднять приложение: `docker compose up --build`;
3. Начать тестирование запросов коллекции.

## Переменные окружения

Рекомендуется использовать следующие переменные:

- `base_url` — базовый URL API;
- `email` — email пользователя;
- `password` — пароль пользователя;
- `access_token` — access token после логина;
- `wrong_email` — email другого пользователя для проверки;
- `wrong_password` — пароль другого пользователя для проверки.

Пример:

```json
{
  "base_url": "http://localhost:8080",
  "email": "user@example.org",
  "password": "password",
  "wrong_email": "wronguser@example.org",
  "wrong_password": "wrongpassword"
}
